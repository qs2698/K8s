- ✅ 로그인/로그아웃 기능 구현 완료 (테스트 계정 사용)

## ⚙️ 환경 설정 (필수)

### 1. Node.js 설치 확인
```bash
node --version  # 18.x 이상이어야 함
npm --version   # 9.x 이상 권장
```

### 2. 프로젝트 클론 후 의존성 설치
```bash
git clone [저장소 URL]
cd KCSPproject
npm install
```

### 3. 개발 서버 실행 확인
```bash
npm run dev
```
브라우저에서 `http://localhost:5173` 접속 확인

## 🎯 통일 사항

### 필수 통일 사항
1. **패키지 관리자**: **npm** 사용 (yarn, pnpm 사용 지양)
2. **Node.js 버전**: **18.x 이상** (팀원 모두 동일 버전 권장)
3. **브랜치 전략**: 
   - `main`: 배포 브랜치 (절대 직접 푸시 금지)
   - `develop`: 개발 브랜치
   - `feature/기능명`: 기능 개발 브랜치
   - `bugfix/버그명`: 버그 수정 브랜치

### 작업 전 필수 체크
1. ✅ 최신 코드 pull 받기: `git pull origin develop`
2. ✅ 의존성 설치: `npm install`
3. ✅ 개발 서버 실행 확인: `npm run dev`
4. ✅ 린트 오류 확인: `npm run lint`

### 작업 후 필수 체크
1. ✅ 코드 정리: `npm run lint` 실행
2. ✅ 커밋 메시지: 명확하게 작성 (예: "feat: 로그인 기능 추가")
3. ✅ PR(Pull Request) 작성: 작업 내용 상세히 설명

## 📝 커밋 메시지 규칙

```
타입: 간단한 설명

상세 설명 (선택사항)
```

### 타입 종류
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅 (기능 변경 없음)
- `refactor`: 코드 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드 과정 또는 보조 도구 변경

### 예시
```
feat: 로그인 페이지 추가

- 사용자 인증 기능 구현
- 테스트 계정 로그인 기능 추가
```
